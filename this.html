    JavaScript "this" Keyword – Complete Mastery (From Basics to Advanced)

    What is "this" Keyword?

    Most Common (WRONG) Understanding:
    "this always points to the object"

    ❌ This is WRONG.

    Correct Definition:
    The value of this is NOT fixed.
    It depends on how the function is called.

    👉 This is the ONLY rule you need to remember: 

    
this → depends on how it is being called

    This was emphasized multiple times in the session. :contentReference[oaicite:0]{index=0}

    ---

    Why "this" is Confusing?

    • It behaves differently in different situations
    • Same code → different output
    • Most tutorials teach it incorrectly
    ---

    5 Types of "this" Behavior

    There are 5 major scenarios:

    • 1. Function Calling
    • 2. Object Calling
    • 3. Constructor Calling
    • 4. Indirect Calling (call, apply, bind)
    • 5. Arrow Function
    ---

    1. Function Calling (Direct Call)

    
function test() {
  console.log(this);
}

test();

    Explanation

    • Function is called directly
    • Not attached to any object

    Output

    
window (global object)

    👉 Rule:

    Direct function call → this = window (global object)
    ---

    2. Object Calling

    
let obj = {
  a: 100,
  fn: function() {
    console.log(this);
  }
};

obj.fn();

    Explanation

    • Function is called using object
    • obj.fn() → depends on obj

    Output

    
{ a: 100, fn: f }

    👉 Rule:

    Object calling → this = object

    Specifically: "this points to the object inside which the method is available" :contentReference[oaicite:1]{index=1}

    ---

    ⚠️ Tricky Case (VERY IMPORTANT)

    
let obj = {
  a: 10,
  fn: function() {
    console.log(this);
  }
};

let x = obj.fn;
x();

    Output

    
window

    Why?

    • Function is stored in variable
    • Now called directly → NOT object call

    👉 Same function, different call → different this

    ---

    3. Constructor Calling

    
function Person(name, age) {
  this.name = name;
  this.age = age;
}

let p1 = new Person("John", 25);

    Explanation

    • Using new keyword
    • Creates new object

    👉 Rule:

    Constructor → this = newly created object
    ---

    4. Indirect Calling (call, apply, bind)

    Purpose

    To manually control this

    ---

    call()

    
let obj1 = { a: 10 };

function test() {
  console.log(this.a);
}

test.call(obj1);

    Output

    
10
    ---

    apply()

    
test.apply(obj1);

    Same as call but:

    • call → arguments separated by comma
    • apply → arguments in array
    ---

    bind()

    
let newFunc = test.bind(obj1);
newFunc();

    Key Difference

    • call/apply → execute immediately
    • bind → returns new function

    👉 From transcript:

    • call/apply → change reference of this
    • bind → creates a copy of function
    ---

    5. Arrow Function (Most Important)

    Golden Rule:
    Arrow function does NOT have its own this.
    It uses its parent's this.
    ---

    Example

    
let obj = {
  name: "John",
  fn: function() {
    let arrow = () => {
      console.log(this);
    };
    arrow();
  }
};

obj.fn();

    Output

    
obj
    ---

    Why?

    • arrow has no own this
    • inherits from parent (fn)
    • fn → called by obj
    ---

    🔥 Important Insight

    Arrow function:

    • ❌ does NOT depend on how it's called
    • ✅ depends on its parent

    From session:

    "Parent ka this = child ka this" :contentReference[oaicite:2]{index=2}

    ---

    🔁 Step-by-Step Thinking (MOST IMPORTANT)

    Teacher's approach:

    1. Go to this
    2. Ask: Why is this executed?
    3. Check function call
    4. Check HOW it is called

    👉 This method never fails. :contentReference[oaicite:3]{index=3}

    ---

    Final Summary

    • this is NEVER fixed
    • Depends on how function is called
    • Direct call → window
    • Object call → object
    • Constructor → new object
    • call/apply/bind → manual control
    • Arrow → parent’s this
