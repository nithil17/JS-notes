    ✨ Template Literals (Backticks)

    Template literals are used to insert variables directly inside strings using backticks (`).

    Teacher’s idea:
    “Instead of using multiple + signs, we can directly embed variables inside a string.”

    🧵 Problem with Normal Strings (Concatenation)

    Using + to join strings becomes messy and hard to read. 
    
let name = "Samarth";
let profession = "teaching";
let city = "New Delhi";

let output = "Hi I am " + name + " " + profession + " in " + city;

console.log(output);

    🧠 Explanation

    - We are joining multiple strings using +
    - Every variable needs separate handling
    - For long sentences → too many + signs ❌

    Output: Hi I am Samarth teaching in New Delhi

    ✅ Solution: Template Literals

    Use backticks (`) and ${} to directly insert variables. 
    
let name = "Samarth";
let profession = "teaching";
let city = "New Delhi";

let output = `Hi I am ${name} ${profession} in ${city}`;

console.log(output);

    🧠 Step-by-Step Flow

    Step 1: Use backticks (` `) instead of quotes
    Step 2: Wherever you need a variable → use ${variable}
    Step 3: JavaScript automatically replaces variables with values

    Output: Hi I am Samarth teaching in New Delhi

    ⚡ Important Rules

    • Template literals work ONLY with backticks (`)
    • Use ${} to insert variables
    • No need for + operator

    🔥 Why Template Literals are Better

    Cleaner, readable, and scalable way to build strings.
    Concatenation Template Literal
    "Hi " + name + " from " + city `Hi ${name} from ${city}`

    🚨 Interview Insight

    Template literals are the preferred modern way (ES6 standard).

    Teacher’s message:
    “Concatenation works, but template literals are the professional way.”
