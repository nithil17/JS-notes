    JavaScript Promises & Async/Await (Teacher Style Explanation)

    What is a Promise? (Teacher Explanation)

    Promise means "vada" (a commitment). If I promise you something, there are only two possible outcomes:

    • ✅ Promise is fulfilled (Resolved)
    • ❌ Promise is not fulfilled (Rejected)

    There is NO third outcome. This is the core mindset behind JavaScript Promises. :contentReference[oaicite:0]{index=0}

    Definition:
    A Promise is an object that represents the eventual completion (resolve) or failure (reject) of an asynchronous operation.
    ---

    Promise Syntax (Step-by-Step)

    
let p1 = new Promise((resolve, reject) => {
    // async work
});

    Teacher explanation:

    • Promise is a constructor function
    • It always starts with new
    • It accepts a callback function
    • The callback has two parameters:
      • resolve → success
      • reject → failure
    ---

    Important Concept (Teacher Words)

    "Resolve and reject are just names. You can write anything, but:

    • First parameter → always behaves like resolve
    • Second parameter → always behaves like reject

    Both are functions."

    ---

    Promise States

    • Pending → initial state
    • Resolved → success
    • Rejected → failure
    ---

    Promise Methods (.then, .catch, .finally)

    
p1
  .then((data) => {})
  .catch((error) => {})
  .finally(() => {});

    Teacher Explanation

    • .then() → runs when promise is resolved
    • .catch() → runs when promise is rejected
    • .finally() → runs in both cases

    "Finally will execute no matter what — resolve or reject." :contentReference[oaicite:1]{index=1}

    ---

    Example with Output

    
let p1 = new Promise((resolve, reject) => {
    resolve(100);
});

p1
  .then((data) => {
    console.log(data);
  })
  .catch((err) => {
    console.log(err);
  })
  .finally(() => {
    console.log("Done");
  });

    Output

    
100
Done
    ---

    Important Rule (VERY IMPORTANT)

    You cannot resolve and reject at the same time.

    👉 The first one executed wins (short-circuiting behavior).

    ---

    Passing Data in Promise

    
resolve("Success Data");
reject("Error Data");

    That data is received inside:

    • .then(data)
    • .catch(error)
    ---

    Promise Chaining (MOST IMPORTANT)

    Instead of nested callbacks (pyramid of doom), we chain promises: 

    
step1()
  .then((res1) => step2(res1))
  .then((res2) => step3(res2))
  .then((res3) => step4(res3))
  .catch((err) => console.log(err));

    Teacher Insight

    "Code grows vertically, not horizontally — no pyramid of doom." :contentReference[oaicite:2]{index=2}

    Async & Await

    ---

    Async Function (Golden Rule)

    
async function test() {
  return 10;
}

    Output

    
Promise { resolved: 10 }

    Teacher Rule

    "No matter what you return → async ALWAYS returns a Promise." :contentReference[oaicite:3]{index=3}

    ---

    Await Concept

    We use await when we want to wait for a promise.

    Rule:
    async can work alone
    await ALWAYS needs async
    ---

    Example

    
async function fetchData() {
  let data = await fetch("API_URL");
  console.log(data);
}
    ---

    What does await actually do?

    Teacher explanation:

    • It blocks the function execution
    • But does NOT block JavaScript completely

    "JavaScript waits for nobody, but await pauses that function." :contentReference[oaicite:4]{index=4}

    ---

    Execution Example

    
console.log(1);

async function test() {
  console.log(10);
  let res = await fetch("API");
  console.log(20);
}

test();

console.log(2);
console.log(3);

    Output

    
1
10
2
3
...response...
20
    ---

    Key Understanding (INTERVIEW GOLD)

    • await pauses only that function
    • global code keeps running
    • async makes function return promise
    ---

    Final Summary

    • Promise → handles async operations
    • .then → success
    • .catch → error
    • .finally → always runs
    • async → always returns promise
    • await → pauses function execution
