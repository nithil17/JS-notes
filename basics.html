    JavaScript Basics

    Introduction to JavaScript

    JavaScript is a programming language used to write logic and instructions that a computer can execute.

    From this point onward, JavaScript becomes the most important part of development. HTML and CSS are for structure and design, but JavaScript is responsible for logic and behavior.

    What Is A Programming Language?

    A programming language is a language used to communicate with a computer and write logical instructions.

    HTML and CSS also give instructions, but they do not allow us to write logic. JavaScript allows decision making, conditions, and dynamic behavior.

    if(userLoggedIn){ showDashboard() }

    Key Properties of JavaScript

    Single Threaded

    JavaScript executes one task at a time.

    Think of a cashier in a shop. Only one person is served at a time.

    Synchronous

    JavaScript executes code line by line in order.

    It does not skip lines. Each line must finish before moving to the next.

    Weakly Typed

    Variables do not have fixed data types.
    let a = 10 a = "hello" a = true

    Dynamically Typed

    The type of variable is decided at runtime.

    JavaScript decides the type when the code runs, not before.

    Interpreted Vs Compiled Languages

    Programming languages are of two types:
    • Compiled Languages
    • Interpreted Languages

    Compiled Languages

    In compiled languages, the entire code is read and checked before execution.

    A compiler goes through the complete code from top to bottom and finds all errors before running the program.

    C, C++, Java -> Compiled Languages
    If there is even one error, the program will not run until all errors are fixed.

    Interpreted Languages

    In interpreted languages, the code is executed line by line.

    Each line is executed one after another. If an error occurs, execution stops immediately and the next lines are not executed.

    Line 1 -> runs Line 2 -> runs Line 3 -> error X Line 4 -> will NOT run
    JavaScript is an interpreted language.

    Key Difference

    Compiled Interpreted
    Entire code runs together Runs line by line
    Errors shown before execution Errors shown during execution
    Faster after compilation Slightly slower

    Instructor Insight

    JavaScript executes code line by line. If an error occurs, execution stops at that point and does not proceed further.

    Variables & Types

    Data Types

    Primitive Data Types

    Primitive data types are the smallest units that cannot be broken further.

    Example analogy:

    Paneer -> Milk -> Milk cannot be broken further -> Primitive

    • Number
    • String
    • Boolean
    • Undefined
    • Null
    • Symbol
    • BigInt

    Non-Primitive Data Types

    Everything in JavaScript is an object.
    let user = { age: 20, isMale: true }

    Variables As Pointers

    According to the instructor, variables in JavaScript do not behave like traditional containers. Instead, they act as pointers (references) to values stored in memory.

    • abc = 10 -> variable points to value 10
    • abc = true -> pointer shifts to new memory location
    • abc = "samarth" -> pointer shifts again
    A variable always points to the latest assigned value, not the previous ones.

    Garbage Collection

    JavaScript automatically removes unused values from memory.

    When a variable stops pointing to a value, and no other variable is using it, JavaScript's garbage collector removes it from memory.

    let a = 10 a = true a = "hello"
    • 10 -> removed (unused)
    • true -> removed (unused)
    • "hello" -> remains
    This prevents memory leakage automatically.

    Advantages And Disadvantages

    • Memory Efficient: Unused values are automatically removed
    • Flexible: Variables can store any type of data
    • Simple Syntax: No need to declare data types
    • Dynamic Behavior: Easy to change values during runtime
    • Beginner Friendly: No strict typing rules
    • Less Control: Developer cannot manually manage memory
    • Runtime Errors: Type issues are detected late
    • Debugging Difficulty: Dynamic typing can cause unexpected bugs

    Variable Nomenclature

    In JavaScript, variables must follow certain naming rules. These rules ensure that the code is valid, readable, and understandable by both the developer and the JavaScript engine.

    • Variable names cannot start with numbers (0-9)
    • Variable names can contain alphabets (a-z, A-Z) and digits (0-9)
    • Only two special characters are allowed: _ and $
    • Reserved keywords cannot be used as variable names
    • Variable names can start with _ or $
    let xyz = 100;
    JavaScript does not require you to define the data type while creating variables. The type is automatically decided based on the value.
    let a = 10 let b = "hello" let c = true let x = 10 x = "text" x = false

    Checking Data Types Using typeof

    In JavaScript, we use the typeof operator to check the type of a variable.

    typeof is used to identify the data type of any variable.
    let str1 = "sam" let str2 = 'vohra' let str3 = `maverick` console.log(typeof str1) console.log(typeof str2) console.log(typeof str3)
    typeof 10 // "number" typeof 10.5 // "number" typeof true // "boolean" typeof "hello" // "string"
    JavaScript does not differentiate between integer and float. Both are considered as number.

    Console & Browser

    Console In JavaScript

    Console is a developer tool that works as a REPL system.

    REPL stands for:

    • Read
    • Evaluate
    • Print
    • Loop

    How Console Works

    When you open the browser console, it continuously waits for your input. This is the Read phase.

    1 + 1

    When you press Enter, the console evaluates the expression. This is the Evaluate phase.

    2

    This is the Print phase. After printing, the console loops back and waits for the next input.

    REPL Flow

    • Console reads your input
    • Evaluates the code
    • Prints the result
    • Loops back to wait for next input

    Examples

    10 + 5 // Output: 15 "Hi" + " JS" // Output: Hi JS true && false // Output: false
    Console is not just for output. It is used to test, debug, and experiment with code in real time.

    Important Clarification

    • Console is a tool
    • JavaScript is the language
    Console does not convert code to machine language. It only helps execute and display results.

    JavaScript In Browser

    JavaScript is the only programming language directly understood by browsers.

    You can run JavaScript directly inside the browser console.

    Console is a developer tool that works as a REPL system (Read, Evaluate, Print, Loop) where JavaScript code can be executed and tested in real time.
