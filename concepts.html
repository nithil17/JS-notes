    JavaScript Core Concepts

    This guide is designed to help you master JavaScript output-based questions commonly asked in interviews. Each concept is explained step-by-step with clear mental models and examples.

    1. Type Coercion

    Definition: Type coercion is when JavaScript automatically converts a value from one type to another to perform an operation.

    Key Mental Model

    • + → prefers string
    • - * / → converts to number
    • Boolean → true = 1, false = 0

    Example 1: String + Number

    console.log("5" + 2);

    Step-by-step:

    • "5" is a string → convert 2 → "2"
    • "5" + "2" = "52"

    Output: "52"

    Example 2: Mixed Operations

    console.log(5 + 2 + "3");
    • 5 + 2 = 7
    • 7 + "3" = "73"

    Output: "73"

    Example 3: Numeric Operators

    console.log("10" - "4" - "3");
    • "10" - "4" = 6
    • 6 - "3" = 3

    Output: 3

    Key Insight: "+" concatenates, while "-", "*", "/" force numeric conversion.

    2. == vs ===

    Definition:
    • == compares values after type conversion
    • === compares value and type without conversion

    Example

    console.log(false == "0");
    • false → 0
    • "0" → 0
    • 0 == 0 → true

    Strict Comparison

    console.log(5 === "5");

    Output: false

    Always prefer === in real-world coding.

    3. Truthy and Falsy Values

    Values that behave as true or false in conditions.

    Falsy Values (ONLY 6)

    false, 0, "", null, undefined, NaN

    Everything Else is Truthy

    [], {}, "0"

    Example

    if ([]) {


console.log("Yes");
}

    Output: Yes

    Arrays and objects are always truthy.

    4. Short Circuiting

    OR (||)

    Returns the first truthy value.

    console.log(null || 0 || "JS");

    Output: "JS"

    AND (&&)

    Returns the first falsy value or last value.

    console.log("Hi" && 0 && "JS");

    Output: 0

    5. Hoisting

    JavaScript moves variable and function declarations to the top before execution.

    var Example

    console.log(a);
var a = 10;

    Internally:

    var a;
console.log(a); // undefined
a = 10;

    let Example

    console.log(a);
let a = 10;

    Output: ReferenceError

    let and const exist in the Temporal Dead Zone.

    6. Functions (Hoisting Behavior)

    Function Declaration

    test();
function test() {
console.log("Hello");
}

    Output: Hello

    Function Expression

    test();
var test = function() {
console.log("Hello");
};

    Output: TypeError

    7. Objects vs Primitives

    Primitives are copied, objects are referenced.

    Primitive Example

    let a = 10;
let b = a;
b = 20;

    a = 10

    Object Example

    let a = {x:10};
let b = a;
b.x = 20;

    a.x = 20

    8. Weird JavaScript Behavior

    typeof

    typeof NaN → "number"
typeof null → "object"

    Comparison Chaining

    console.log(3 > 2 > 1);
    • 3 > 2 → true
    • true → 1
    • 1 > 1 → false

    9. Closures

    A closure is when a function remembers variables from its outer scope even after execution. 
    function outer() {

let x = 10;
return function() {
console.log(x);
};
}

    Even after outer() finishes, x is remembered.

    10. var vs let in Loops

    var

    for (var i = 0; i < 3; i++) {


setTimeout(() => console.log(i));
}

    Output: 3 3 3

    let

    for (let i = 0; i < 3; i++) {


setTimeout(() => console.log(i));
}

    Output: 0 1 2

    Final Interview Strategy

    • Check operator type (+ vs -)
    • Check type coercion
    • Check == vs ===
    • Check truthy/falsy
    • Check scope (var/let)
    • Check closures / async
