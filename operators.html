    Operators in JavaScript

    Operators are symbols used to perform operations like addition, comparison, and logic in JavaScript.

    As explained in class:
    “Operators are just tools or symbols used to perform operations.”

    Arithmetic Operators

    Used for basic mathematical calculations. 
    
+  → Addition
-  → Subtraction
*  → Multiplication
/  → Division
%  → Modulus (Remainder)
** → Power

    Power Operator (**)

    Power means multiplying a number by itself multiple times. 
    
console.log(2 ** 3);

    2 × 2 × 2 = 8

    Output: 8

    Modulus Operator (%)

    Modulus gives the remainder after division. 
    
console.log(25 % 3);

    25 ÷ 3 = 24 remainder 1

    Output: 1

    More Examples

    
5 % 3 → 2
6 % 3 → 0
7 % 3 → 1
3 % 2 → 1
3 % 3 → 0

    Important Note (JavaScript Numbers)

    JavaScript does NOT have separate integer or float types. Everything is a number. 
    
let a = 10;
let b = 20;

console.log(a / b);
    Output: 0.5

    Unlike C++, JavaScript does NOT convert it to 0.

    Comparison Operators

    Used to compare values. Output is always true or false. 
    
10 > 3   → true
3 > 10   → false
5 >= 5   → true

    Tricky Concept: Type Coercion

    JavaScript automatically converts true → 1 and false → 0 during comparisons. 
    
console.log(3 > 3 > 2);

    Step-by-step:
    3 > 3 → false → becomes 0
    0 > 2 → false

    Output: false

    Equality Operators

    Used to check if two values are equal.

    Double Equals (==)

    Checks only value, ignores type. 
    
console.log(10 == "10");
    Output: true

    Triple Equals (===)

    Strict equality → checks both value AND type. 
    
console.log(10 === "10");
    Output: false

    Teacher trick:
    == → Mother (less strict)
    === → Father (strict)

    Logical Operators

    OR (||)

    If ANY condition is true → result is true. 
    
let marks = 45;
let attendance = 60;

if(marks >= 33 || attendance >= 75){
  console.log("Pass");
}
    Output: Pass

    Even if one condition is true, it passes.

    AND (&&)

    All conditions must be true. 
    
if(marks >= 33 && attendance >= 75){
  console.log("Pass");
}else{
  console.log("Fail");
}
    Output: Fail

    🔥 Short Circuiting (Very Important Concept)

    Short circuiting means: as soon as the final result is known, JavaScript stops checking further conditions.

    This happens in logical operators (|| and &&).

    Teacher’s line:
    “As soon as the final resultant is known, the further operation stops.”

    Short Circuiting with OR (||)

    In OR (||), only ONE true condition is enough. 
    
let marks = 45;

if(marks >= 33 || attendance >= 75){
  console.log("Pass");
}

    🧠 Step-by-Step Flow

    Step 1: Check first condition → marks >= 33
    45 >= 33 → true ✅

    Step 2: Now JavaScript already knows the final result:
    In OR → if one condition is true → result is TRUE

    Step 3: So it does NOT check attendance ❌

    Output: Pass

    ⚠️ Even if attendance is not defined, no error will come because it is never checked.

    Short Circuiting with AND (&&)

    In AND (&&), ALL conditions must be true. 
    
let marks = 15;

if(marks >= 33 && attendance >= 75){
  console.log("Pass");
}else{
  console.log("Fail");
}

    🧠 Step-by-Step Flow

    Step 1: Check first condition → marks >= 33
    15 >= 33 → false ❌

    Step 2: In AND → if ANY condition is false → result is FALSE

    Step 3: So JavaScript stops immediately and does NOT check attendance

    Output: Fail

    ⚠️ Even if attendance is missing, no error occurs because it was never evaluated.

    🚨 Important Interview Case

    
let marks = 45;

if(marks >= 33 || attendance >= 75){
  console.log("Pass");
}

    If attendance is not declared:

    Normally → accessing undefined variable → ERROR ❌
    But here → NO ERROR ✅

    Because:
    First condition is true → final result known → second condition skipped

    ⚡ Summary

    • OR (||): Stops when it finds first TRUE
    • AND (&&): Stops when it finds first FALSE
    • Main Rule: Once result is decided → execution stops
